Programming in Ximview

It would sometimes be convenient to call Ximview from user program to allow display of new images or analysis of a sequence of images. A direct attempt to do this fails as ximview is designed to run as an unblocked widget in parallel with other IDL processes. Instead, user programs invoking Ximview have to be run within the ximview event loop. The basic idea is that the program is initiated from a "Run program" menu option, which prompts the user for the name of the program and initial data. The program then executes a start-up sequence and stores its parameters in a structure, which is returned to ximview. The name of the program and the control structure are saved by ximview. Subsequent interaction is via the programmable buttons labelled "A", "B", "C": pressing any of theses calls the program with the button label and control structures as arguments. This system is currently under development and so far the available programs are hard-coded under Analysis -> Catalogues -> Process Catalogue.
